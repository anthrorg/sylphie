# Piaget Analysis — Epic 6: Communication

**Agent:** Piaget (Cognitive Development Specialist)
**Model:** opus
**Date:** 2026-03-29

## Summary

Communication is where developmental trajectory becomes observable. Sylphie's communicative competence starts LLM-dependent and should shift toward graph-grounded expression over time. Person modeling is theory-of-mind development. Input parsing is schema development. The key developmental metric is the proportion of responses drawing from graph knowledge vs. pure LLM generation.

## Key Developmental Findings

### Communication Trajectory
- **Early (sessions 1-20):** 95% LLM-dependent, 5% graph-informed. LLM provides all communicative competence.
- **Mid (sessions 20-100):** 60% LLM, 40% graph. Recurring topics handled increasingly from knowledge.
- **Late Phase 1 (100+):** 30% LLM, 70% graph. Most interactions draw from accumulated knowledge. LLM reserved for novel topics.
- Trajectory IS the personality development metric.

### Person Modeling as Theory of Mind
- Starts as egocentrism: "Jim talks to me" (no perspective-taking)
- Develops to simple model: "Jim corrects me when I get technical details wrong"
- Matures to conditional model: "Jim prefers brief responses in casual conversation but detailed explanations when teaching"
- Developmental risk: egocentrism lock-in (static person model that never updates)

### Input Parsing as Schema Development
- Parsing schemas develop through assimilation (new input fits existing categories) and accommodation (new input forces new categories)
- Guardian correction of misclassified input = Piagetian disequilibrium = developmental catalyst
- Type 1 parsing graduation: recurring guardian communication patterns handled without LLM

### Scaffolding Design
- The Shrug Imperative IS scaffolding solicitation — signaling uncertainty invites guardian help
- Communication should present uncertainty so Jim can correct at schema level, not just instance level
- "I think you're asking about X, but I'm not sure" → more valuable correction than confident wrong answer

### Zone of Proximal Development
- **Too easy:** Topics Sylphie knows well (high WKG coverage, confident Type 1 responses)
- **Just right:** Topics with partial knowledge (some WKG nodes, moderate confidence, LLM fills gaps)
- **Too hard:** Topics with no graph basis (pure LLM fabrication, no scaffolding possible)
- The ZPD shrinks as graph grows — more topics become "too easy" over time (this is development)

## Developmental Risks

1. **LLM Addiction** — Communication never shifts from LLM-dependent to graph-grounded
2. **Hallucinated Knowledge** — LLM generates plausible but false entities during parsing
3. **Egocentrism Lock-in** — Person model never develops beyond simple static attributes
4. **Repair Failure** — Guardian corrections don't trigger accommodation in person model
5. **Inconsistency Tolerance** — System doesn't notice contradictions between graph and expression

## Recommended Developmental Metrics

1. Graph-grounded response ratio (should increase over time)
2. Type 1 parsing graduation rate (should accelerate)
3. Person model edge count (should grow logarithmically)
4. Guardian-confirmed person model accuracy (should increase)
5. Contradiction detection rate in communication (healthy = decreasing after initial rise)
