# Update Canon

Propose and apply a change to the CANON document. Requires Jim's explicit approval.

## Usage

```
/update-canon "Add detailed drive cross-modulation rules to appendix A.1"
/update-canon "Revise Phase 1 scope to include basic Planning"
```

## When to Use

- When a CANON gap is identified during planning or implementation
- When Jim requests a CANON change
- When an appendix section needs to be filled in

---

## Workflow

### Phase 1: DOCUMENT CHANGE

1. Read current `wiki/CANON.md`
2. Draft the proposed change with:
   - What currently exists
   - What the proposed change is
   - Why the change is needed
   - Impact on existing plans/implementations

### Phase 2: JIM'S APPROVAL

Present the proposal to Jim. **The CANON is immutable unless Jim explicitly approves.**

### Phase 3: APPLY

If approved:
1. Apply the change to `wiki/CANON.md`
2. Update any affected planning documents
3. Notify relevant agents of the change

---

## Key Rules

- NEVER modify the CANON without Jim's explicit approval
- Changes are documented with rationale
- Impact assessment is mandatory
