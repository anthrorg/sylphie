DOCKERFILE (root, single multi-stage):
- Build order: shared → prisma generate → drive-engine → decision-making →
  learning + planning (parallel) → @sylphie/app → vite frontend
- Production stage installs --production deps, copies dist and frontend/dist,
  exposes 3000, CMD ["node","apps/sylphie/dist/main.js"].
- Note: drive-server is NOT included in the image. Production deployment runs
  only the main NestJS app + WebSocket-served frontend on a single port.
  This is a Railway-style single-process deployment; the dual-process
  drive-isolation only applies in development.

PRISMA SCHEMA (packages/shared/prisma/schema.prisma):
  ONE model, User: id (uuid PK), username (unique), passwordHash @map(password_hash),
  approved (default false), isGuardian @map(is_guardian) (default false),
  createdAt @map(created_at) default now(). Table: users.
  Migrations: add_user_approved_column, add_user_is_guardian_column.
  Everything else (drives, rules, events) lives outside Prisma; Prisma is auth-only.

NESTJS MODULES (all @Global()):
- PrismaModule  → PrismaService (extends PrismaClient; URL synthesized from
  postgres config runtimeUser/runtimePassword; 5×3s connect retries).
- TimescaleModule → TimescaleService wraps pg.Pool; methods query<T>, getClient,
  withTransaction(BEGIN/COMMIT/ROLLBACK), 5×3s connect retries.
- Neo4jModule.forRoot/forRootAsync registers NEO4J_INSTANCE_CONFIG +
  Neo4jService. Neo4jService manages Map<Neo4jInstanceName,Driver>; methods
  getDriver, getSession(name, mode='WRITE'); verifyConnectivity()
  with 5×3s retries.

INJECTION TOKENS (no module registers):

GATEWAYS (WebSockets via WsAdapter):

    - In onModuleInit subscribes to communication.delivery$; broadcasts thinking_indicator:false then full DeliveryPayload

WkgQueryService (816L):
  Manages all four KG instances (WORLD/SELF/OTHER/PKG).
  onModuleInit creates indexes for WORLD/SELF/OTHER:
    CONSTRAINT ap_id_unique on ActionProcedure.id
    INDEX ap_node_id, ap_confidence, conversation_session, insight_node_id,
          drive_name, cobeing_label, entity_label, entity_node_id
    FULLTEXT INDEX kg_label_fulltext on (Entity|ActionProcedure|CoBeing|Drive|
                                         Insight|Conversation|Attribute) ON EACH [n.label]
  Migration: SET n:Entity for unlabeled nodes with non-null label property.
  PKG snapshot cached 60s, EXCLUDES :CodeBlock nodes (heavy bodyText) and
  strips bodyText/args/properties props, capped 8000 nodes / 15000 edges.
  Search uses kg_label_fulltext with prefix wildcard.
  Neighborhood: N-hop ego-graph capped at 500 nodes, hops clamped 1-3.

WkgBootstrapService (398L):
  bootstrap() (idempotent):
    1. CoBeing anchor {node_id:'cobeing-self', label:'Sylphie',
       description:'A young mind learning about the world through direct
       experience', phase:'1.5', version:'0.1.0', provenance:INFERENCE, conf:1.0}
    2. 12 :Drive nodes (one per DriveName), category=core|complement,
       provenance:INFERENCE, conf:1.0
    3. Self KG bootstrap: same CoBeing anchor in SELF + 3 Attribute facts
       name='Sylphie', identity='a developing cognitive being',
       phase='1.5 — early development'
  resetWorldOnly:
    DETACH DELETE all WORLD nodes; UPDATE events SET has_learned=false
    WHERE type IN ('INPUT_RECEIVED','INPUT_PARSED'); TRUNCATE
    reflected_sessions, synthesized_insight_pairs CASCADE; re-bootstrap.
    SELF/OTHER/tensor pipeline preserved.
  resetAndBootstrap (full):
    DETACH DELETE all 3 KGs; TRUNCATE learned_patterns, voice_patterns,
    sensory_ticks, events, reflected_sessions; opens own pg.Pool to
    TRUNCATE proposed_drive_rules CASCADE; outcomeReporter.resetDriveState();
    clears in-memory: latentSpace, sensoryPrediction, voiceLatentSpace,
    conversationHistory, personModel, tickSampler, episodicMemory; re-bootstrap.

CommunicationService (1166L):
  Sits between Decision Making executor and ConversationGateway. In onModuleInit
  subscribes to decisionMaking.response$.

  Pipeline per response:
    1. sanitizeResponseText: strips [GROUNDED]/[ASSISTED]/[UNKNOWN] tags,
       [INTENT:]/[ENTITY:]/[THOUGHT:] tags, tool-call leakage like
       [person_query …], leading dashes/em-dashes.
    2. Log RESPONSE_GENERATED to TimescaleDB.
    3. checkTheaterProhibition (FLAG-ONLY currently — anxiety>0.7 + non-empty
       text triggers debug log; does not block).
    4. Voice cache lookup via VoiceLatentSpaceService:
       valence = 0.5 + (positive - negative) * 0.25 where
         positive = satisfaction + curiosity*0.5
         negative = anxiety + sadness + guilt*0.5
       HIT: serve cached audioBase64. MISS: call TtsService and store result.
    5. Emit DeliveryPayload {type:'cb_speech', text, turnId, audio?, isGrounded,
       arbitrationType, latencyMs, llmCalled (=type2), costUsd:0 ('Local Ollama'),
       knowledgeGrounding}
    6. Log RESPONSE_DELIVERED.
    7. addAssistantMessage to ConversationHistoryService.
    8. Store in pendingTurns Map (cap 50) for guardian feedback correlation.
    9. reportBasicOutcome → decisionMaking.reportOutcome with computed
       driveEffectsObserved (delta between preExecutionDriveSnapshot and postSnapshot).

  parseInput: classifies (GREETING|QUESTION|COMMAND|EMOTIONAL_EXPRESSION|
    GUARDIAN_FEEDBACK|STATEMENT) via regex. extractEntities = capitalized words >1.
    detectGuardianFeedback: yes/correct/exactly/good=confirmation, no/wrong=correction.
    Logs INPUT_RECEIVED + INPUT_PARSED. addUserMessage to ConversationHistory.
    setActivePerson + recordInteraction.
    Fast Fact Extraction: extractFactsFromText returns ExtractedFact[] with
      target='speaker'|'sylphie'. writeFastFacts in parallel:
        speaker → personModel.writeFact(OKG) + writeFactToWkg(WORLD)
        sylphie → writeFactToSelfKg(SELF) + writeFactToWkgCoBeing(WORLD)
    factKeyToRelType: name→HAS_NAME, identity→IDENTIFIES_AS, likes→LIKES,
      occupation→WORKS_AS, location→LIVES_IN, age→HAS_AGE, else HAS_<KEY>.
    detectGuardianTeaching: regex patterns ("you should learn", "I want you to",
      "learn how to", "practice", "work on", "you need to", "plan how to",
      "figure out how to", "get better at"). Writes

THE 12 DRIVES (packages/shared/src/types/drive.types.ts):
  DRIVE_INDEX_ORDER 0-11 (strictly canonical):
    0: SystemHealth     - composite of 1+2+3 / 3
    1: MoralValence     - ethical alignment, action-driven via SelfCorrection + guardian
    2: Integrity        - prediction reliability, anxiety cross-modulation
    3: CognitiveAwareness - LLM cost pressure (Type 2 cost)
    4: Guilt            - event-only via guardian correction outcomes
    5: Curiosity        - exploratory, satisfied by information gain
    6: Boredom          - engagement deficit, primary conversation relief
    7: Anxiety          - stress, suppresses curiosity, amplifies failures
    8: Satisfaction     - relief reservoir from successful outcomes
    9: Sadness          - decaying negative affect
   10: Focus            - attention, relieved through structured action
   11: Social           - engagement need, primary conversation relief
  CORE_DRIVES = [0,1,2,3]; COMPLEMENT_DRIVES = [4..11]

  PER-DRIVE ACCUMULATION RATES (per tick):
    SystemHealth, MoralValence, Integrity, CognitiveAwareness, Guilt: 0.0 (event/composite-driven)
    Curiosity:    0.0012 (~10 min to fill from 0.3)
    Boredom:      0.0015 (~7 min from 0.4)
    Anxiety:      0.0003 (~44 min from 0.2)
    Satisfaction: 0.0    decay -0.0009 (~18 min)
    Sadness:      0.0    decay -0.0006 (~28 min)
    Focus:        0.0    decay -0.0006 (~28 min)
    Social:       0.0009 (~9 min from 0.5)
  Decay only applies when current > 0 (drive should not be pushed below equilibrium).

TICK RATE: DRIVE_ENGINE_TICK_INTERVAL_MS = 1000 (1Hz)
  (Many comments still reference an aspirational 100Hz; actual is 1Hz,
   "appropriate for a motivational system where drives change over minutes,
   not milliseconds".)
  Drift compensation via setTimeout, MAX_TICK_DRIFT_MS=100.
  Auto-save checkpoint every 60 ticks → drive_state_checkpoint UPSERT.

TICK SEQUENCE (DriveEngine.tick):
  1. Drain outcome queue
  2. Self-evaluation if due (every 10 ticks, SELF_EVALUATION_INTERVAL_TICKS)
  3. Apply accumulation/decay
  4. Apply queued outcomes
  5. Cross-modulation
  6. Clamp all drives
  7. Compute totalPressure (sum of positives only)
  8. Update opportunity detector
  9. Build snapshot, publish to transport (WS)
  10. Decay opportunities every tick
  11. Emit top opportunities (max 5/cycle)
  12. Auto-save checkpoint (every 60 ticks)
  13. Advance tick counter

RULE ENGINE (rule-engine.ts):
  Loads drive_rules from Postgres at startup; reloads every RULE_RELOAD_INTERVAL_MS=60000.
  Trigger pattern DSL: "action_success AND anxiety > 0.7" or "prediction_hit OR prediction_miss".
    Operators: > < = >= <= !=
  Effect DSL: += -= *= = (multi-effect via ;)
  RULE_CONFIDENCE_THRESHOLD = 0.3 (rules below ignored).
  LRU cache size 500, key = eventType|drive_state_hash (only Anxiety/Satisfaction/
    Guilt/CognitiveAwareness hashed for compactness).
  When no rule matches, falls back to ACTION_TYPE_DEFAULTS (constants/rules.ts):
    Categories: ConversationalResponse, GuardianEngagement, SocialComment,
                LearnedResponse, KnowledgeQuery, SocialInquiry, VisualInquiry,
                SelfCorrection, GuardianTeaching, plus four metadata-scaled
                sensory types (UndiscoveredObjectPressure, UnknownPersonPressure,
                SensoryPrediction, ScenePrediction).
    PRIMARY_RELIEF = -0.367, SECONDARY_RELIEF = -0.183 (tuned so ~30 conversation
                                                       responses move primary
                                                       drives 1.0 → -10.0).
    OUTCOME_DEFAULTS: guardian_confirmation→Satisfaction+0.15, MoralValence-0.10,
                                            Anxiety-0.05.
                     guardian_correction→Guilt+0.15, Satisfaction-0.10.
  When defaults are used, INSERT into proposed_drive_rules with proposed_by='DRIVE_ENGINE_DEFAULT'
    for guardian to debate (deduplicated).

GUARDIAN WEIGHTING (after rule effects, before contingencies):
  guardian_confirmation × 2.0
  guardian_correction × 3.0
  algorithmic × 1.0  (CANON Standard 5)

CROSS-MODULATION (5 typed rules, ordered):
  1. satisfaction-suppresses-boredom (multiplicative, threshold 0.6, coef 0.03)
  2. anxiety-amplifies-integrity (additive, threshold 0.7, coef 0.0012)
  3. system-health-amplifies-anxiety (additive_gap, threshold 0.7, coef 0.003)
     uses PREVIOUS tick's SystemHealth (no feedback loop)
  4. boredom-amplifies-curiosity (additive_gap, threshold 0.6, coef 0.003)
  5. guilt-suppresses-satisfaction (multiplicative, threshold 0.4, coef 0.03)
  THEN: SystemHealth = mean(MoralValence, Integrity, CognitiveAwareness)

5 BEHAVIORAL CONTINGENCIES (applied after theater check passes):
  1. Satisfaction Habituation: per-actionType consecutive-success curve
     1st=+0.05, 2nd=+0.04, 3rd=+0.03, 4th=+0.02, 5th-10th=+0.01, 11th+=+0.005.
     Failure resets counter.
  2. Anxiety Amplification: when anxiety > 0.7 + negative outcome → 1.5×
     positive deltas. + positive outcome → anxiety -= 0.10 relief.
  3. Guilt Repair: keyword + behavioral-change detection. Schedule:
     ack+change=-0.30, change-only=-0.15, ack-only=-0.10. Tracks last 10
     errors with 15-min timeout.
  4. Social Comment Quality: socialCommentTimestamp + guardian response
     within 30s → social-=0.15, satisfaction+=0.10. Comments buffered 60s.
  5. Curiosity Information Gain: relief = newNodes*0.05 + confidenceDeltas*0.10
     + resolvedErrors*0.15 (returned negative).

CONSOLIDATION PIPELINE (per event, MAX_EVENTS_PER_CYCLE=5):

  Step 2: fetchUnlearnedEvents — SELECT WHERE has_learned=false AND type IN
    ('INPUT_RECEIVED', 'INPUT_PARSED'). Partial index idx_events_unlearned ON
    events (timestamp ASC) WHERE has_learned=false.

  Step 3 (Upsert Entities): extractTitleCasedTokens (regex, no LLM). Two-pass scan:
    distinguishes proper nouns from sentence-initial title-casing. Filters
    STOPWORDS (~95 words) and TEMPORAL_WORDS. Compound entities ("New York City"
    → 1 entity). Cap 20 per event. INPUT_PARSED bypasses regex and uses
    payload.entities directly. MERGE :Entity {label} with provenance per event
    type (SENSOR for INPUT_*, GUARDIAN for guardian events). ON MATCH only RAISES
    confidence.

  Step 3b (Typed Edges): parseTriples runs 10 regex patterns ("I like X",
    "I work at Y", etc). resolveEdgeType maps key to FACT_KEY_TO_EDGE_TYPE
    or generic HAS_<KEY>; favorite_* → HAS_FAVORITE. Each writes via MATCH...
    MERGE (a)-[r:${type}]->(b) with r.refined_from='STRUCTURED'. Returns
    typedPairs Set so co-occurrence skips them.

  Step 4 (RELATED_TO): buildSentencePairs restricts pairing to entities in
    same sentence (split on /(?<=[.!?])\s+/). Skips already-typed pairs.
    Cap 10 pairs. UNWIND batch INSERT MERGE :RELATED_TO. Provenance LLM_GENERATED.

  Step 5 (Conversation Entry): :Conversation {node_id:'conv-<8>'}, content
    truncated 500 chars, provenance SENSOR conf 0.40. MENTIONS edges to entities.

  Step 6 (CAN_PRODUCE Edges): extractSignificantPhrases — bigram + trigram
    sliding window (MIN_TOKEN_LENGTH=3), MAX_PHRASES=15. Each phrase →
    :Word node (LLM_GENERATED 0.35). (:Conversation)-[:CAN_PRODUCE]->(:Word)
    edges (INFERENCE 0.30).

  Step 7 (LLM Edge Refinement, RefineEdgesService):
    Phase 1 — Heuristic (deterministic): classifyByHeuristic against verb
      patterns DISLIKES (intentionally before LIKES), LIKES, KNOWS, WORKS_AT,
      LIVES_AT, CREATED, OWNS. (USES intentionally omitted to avoid "uses
      the bathroom" false positive.) Confident → updateEdgeType refined_from='HEURISTIC'.
    Phase 2 — LLM: only edges where heuristic !confident. If LLM null/unavailable
      (Lesion Test) → skip; edges remain RELATED_TO. gatherPersonContext queries
      last 5 INPUT_PARSED with title-cased single-word labels. Tier 'medium', temp 0.3.
      parseRefinements validates VALID_TYPE_RE = /^[A-Z][A-Z0-9]*(?:_[A-Z0-9]+)*$/,
      length cap 40. ANY well-formed UPPER_SNAKE_CASE accepted.
    Update mechanics: Neo4j cannot rename rel types in place — MERGE new typed
      rel with refined_from='LLM', then DELETE original RELATED_TO.

  Step 7b (Contradiction Detection): ANTONYM_MAP currently only LIKES↔DISLIKES.
    Cypher MATCH (a)-[:antonym]-(b) bidirectional. If found, MERGE :CONTRADICTS
    edge with claim, existingFact, claimConfidence, existingConfidence, conf:0.50,
    event_id, session_id. Consumed by ContradictionScannerService in decision-making.

  Step 2b: markAsLearned UPDATE has_learned=true. Always called even on
    exception (poison-event safeguard).

CONFIDENCE DECAY (confidence-decay.service.ts):
  ACT-R style: new_conf = old_conf - decayRate * log(hoursSince + 1)
  Per-provenance rates: SENSOR 0.05, GUARDIAN 0.03, LLM_GENERATED 0.08, INFERENCE 0.06
  MIN_HOURS_BEFORE_DECAY = 1.0
  pruneOrphanedNodes: only :Entity nodes below 0.10 with NO edges. Excludes
    Drive, CoBeing, ActionProcedure, Conversation, Insight, Word.

CATASTROPHIC INTERFERENCE PREVENTION:
  No EWC, no replay buffer in Learning. Protection model:
  - MERGE upsert only RAISES confidence on match
  - ACT-R decay penalizes old facts; per-provenance rates protect GUARDIAN
  - Pruning only removes orphaned low-confidence non-structural nodes
  (wiki/researchedIdeas/2026-04-27-ewc-real-fisher-computation.md exists in
  git status — research roadmap; not in this package.)

CONVERSATION REFLECTION (conversation-reflection.service.ts):
  Every 5 min. Eligibility: not in reflected_sessions, MAX(timestamp) <
    NOW - 10min, COUNT >= 4 events. Oldest first, LIMIT 1.
  reflected_sessions table: session_id PK, reflected_at, insights_created,
    edges_created.
  Sets up :Insight nodes (provenance INFERENCE base 0.30) with DERIVED_FROM
    (Conversation) and REVEALS (Entity) edges. discovered_by='REFLECTION' on
    new entity edges.
  6 insight types: DELAYED_REALIZATION, MISSED_CONNECTION, IMPLICIT_INSTRUCTION,
    CONTRADICTION, THEMATIC_THREAD, TONAL_SHIFT.
  Tier 'medium', temp 0.3, max 1536 tokens.
  computeGroundedConfidence: scales by revealsCreated/attemptedReveals,
    floor 0.05, grounded=ratio>0. No entities → base preserved.

CROSS-SESSION SYNTHESIS (cross-session-synthesis.service.ts):
  Every 30 min. Cypher matches insights from different sessions REVEALing same Entity.
  CONFABULATION GUARD: parseSynthesisResponse verifies CITES field contains both

  Synthesis :Insight {is_synthesis:true, source_sessions[]} with SYNTHESIZES
    edges to source insights.
  Confidence: mean(source confs) + (sharedCount-1)*0.02 capped 0.10, floored 0.30,

    Change.hash), 10 indexes.

  Project cache cleared every 5 batches (memory bound).
  Relationship pass: CALLS, USES_TYPE, EXTENDS, IMPLEMENTS, INJECTS edges
    (each individual edge attempt try/catch).
  Integrity check: 6 parallel checks (duplicates, missing props, orphaned IMPORTS,
    wrong-label CONTAINS, missing BELONGS_TO chain, duplicate Types).
  Sync cursor: writes git rev-parse HEAD to .last-sync-commit.

SYNC PIPELINE (sync-pipeline.ts):
  getChangedFiles diff against .last-sync-commit; getDeletedFiles.
  Parse + computeChangeset + buildMutations + applyMutations (one transaction).
  logChange records :Change. Integrity check gates cursor advancement —
  if checks fail, cursor stays in place for retry.

MODELS:
  GlobalModel ("Brainstem"):
    Input dim 1561 = EMBEDDING_DIM(768) + DRIVE_VECTOR(12) + DRIVE_DELTAS(12)
                   + 1 + EMBEDDING_DIM(768).
    Architecture: Dense 1561→512 ReLU → 256 ReLU → [32 softmax action_bias,
                   2 sigmoid urgency+novelty]. ~450K params.
    TF Keras path + NumPy fallback (Xavier seed 0xBEEF).
  PanelModel (4 instances): NumPy-only ~100K each.
    Input → 256 ReLU → 128 ReLU → [32 softmax action_bias, 1 sigmoid confidence,
    8 tanh domain_signal]. Per-panel seed = hash(name) & 0xFFFFFFFF.
    Panels:
      drive (input 1681) — extra: drive history 10×12=120
      decision (input 1566) — extra: top-5 latent match scores
      learning (input 1575) — extra: 14 features (10 MAE + 4 novelty)
      planning (input 1569) — extra: 8 opportunity-queue features
  ConvergenceModel: NumPy ~10K. Input 160 = 5×32 (1 global + 4 panels).
    Dense 160→64 ReLU → [1 sigmoid divergence, 4 sigmoid panel adjustments].
    use_learned flag defaults False; heuristic path = pure cosine similarity
    averaged across panels.
  DeliberationSystem: 3 DeliberationPipeline clones (pragmatist, conservative,
    advocate, seeds 0xD1A6/0xC0A5/0xAD10), same arch as global. SynthesisModel:
    96→64→[32 softmax bias, 3 softmax pipeline_weights, 1 sigmoid confidence].
    Total ~1.36M params.
  Total cognition params dynamically computed (~2.22M actual; comment says 5.5M).

BOOTSTRAP MODES (BootstrapTracker):
  shadow → audit: total comparisons across categories >= 100
  audit → partial: at least one category graduated (>=20 samples in window AND
                   agreement >= 0.85)
  partial → full: overall agreement >= 0.90 AND >= 3 graduated categories

ACTION VOCABULARY (ActionVocabulary, training/trainer.py):
  32-slot action space. Index 0 = "unknown", index 31 = "shrug" reserved.
  Indices 1-30 filled in arrival order. When full, new categories collapse to
  "unknown" rather than evict shrug.
  Categories normalized to lowercase before comparison.
  NestJS sends PascalCase like "ConversationalResponse".

TRAINING (online, daemon thread):
  _MIN_BUFFER_SIZE=10, _BATCH_SIZE=32, replay_fraction=0.5.
  Loop:
    1. Wait until buffer >= 10
    2. Sample batch (mixed: 50% random + 50% recent)
    3. _train_step: forward + cross-entropy on action head + Adam update
    4. Increment _training_steps; log every 100
    5. Checkpoint when steps % CHECKPOINT_INTERVAL=1000 AND >=60s since last
    6. _stop_event.wait(0.01) between iterations
  TRAINER ONLY UPDATES NUMPY GLOBAL MODEL — skips TF and other panels.
  Aux head (urgency/novelty) has zero gradient (not supervised during bootstrap).
  EWC penalty added but currently zero (uniform Fisher, never set_reference'd).

DataBuffer ring buffer:
  Capacity REPLAY_BUFFER_SIZE=100,000 (env-configurable).
  threading.Lock. List fields converted to np.float32 on insertion.
  sample_batch returns mixed batch.

CHECKPOINTING:
  Each model serializes independently. Atomic save (.tmp → os.replace).
  GlobalModel.save: .h5 (TF) or global_model_np.npz (NumPy).
  Other models: panel_<name>.npz, convergence_model.npz, delib_<name>.npz,
    synthesis_model.npz.
  Loaders tolerate corruption (try/except, keeps initialized weights).
  Foundation snapshots at WEIGHTS_DIR/foundation/ for "Society of Mind forking."

EWC / FISHER (training/replay.py):
  EWCRegularizer is a STUB. _compute_uniform_fisher returns all-ones
  (= L2 weight anchoring). set_reference NEVER CALLED → EWC is currently a no-op.
  Catastrophic interference prevention is declared (intent) but not realized.

NOTABLE STUBS / NON-FUNCTIONAL:
  - per_category_confidence dict initialized empty, never written.
  - MAX_INFERENCE_TIMEOUT_MS=50 defined but not enforced (no watchdog).
  - boost_salience intervention: not implemented on sidecar.
  - reinforce/correct endpoints are TODO stubs.
  - per-model freeze "not yet implemented".
  - ConvergenceModel.use_learned defaults False; flip happens nowhere
    automatically — same trapdoor as EWC.

SECTION 16 — CROSS-CUTTING ARCHITECTURAL OBSERVATIONS
==================================================================

1. PRINCIPLE: Drive Isolation as Process Boundary
   Drive engine is a separate WebSocket-served process. Main app cannot
   introspect rules. RLS verification at startup ABORTS if sylphie_app can
   UPDATE/DELETE drive_rules. The single-client lock + no-RPC-for-rules
   architectural enforcement implements CANON Standard 6 (No Self-Modification
   of Evaluation).

2. PRINCIPLE: Theater Prohibition at the Type Level
   All SylphieEvent carry mandatory driveSnapshot. ActionOutcomePayload requires
   theaterCheck. If theatrical, drive engine applies ZERO reinforcement.
   Communication's checkTheaterProhibition is currently FLAG-ONLY (logs but
   doesn't block) — full enforcement is in the drive engine.

3. PRINCIPLE: Lesion Test as First-Class Architectural Constraint
   Every LLM-dependent code path checks isAvailable() and degrades gracefully:
   - Learning Phase 2 edge refinement: edges remain RELATED_TO
   - Communication WHO_AM_I trigger: falls back to plain fact list
   - Planning proposal: template fallback (2-step WKG_QUERY + LLM_GENERATE)
   - Tensor inference: returns null → LLM-only path

4. PRINCIPLE: Provenance + Confidence Discipline
   Every node/edge carries provenance_type. ON MATCH only RAISES confidence.
   ACT-R decay penalizes old facts; per-provenance rates protect GUARDIAN-taught
   most strongly. Confidence ceiling 0.60 until guardian confirms;
   Type 1 graduation requires 0.80 + MAE<0.10.

5. PRINCIPLE: Event Backbone (TimescaleDB) for Cross-Subsystem Communication
   Subsystems do not directly inject each other. Communication writes
   GUARDIAN_TEACHING_DETECTED → Planning polls → Planning writes PLAN_CREATED
   → Decision Making outcome evaluator picks up → Drive Engine receives
   ACTION_OUTCOME. EVENT_BOUNDARY_MAP enforces compile-time ownership.

6. PRINCIPLE: Three-Layer Latent Spaces (Hot/Warm/Cold)
   Pattern repeated 3 times:
   - VoiceLatentSpaceService: in-memory 500 / pgvector / archival
   - FaceSnapshotService: centroids / face_embeddings / OKG snapshots
   - LatentSpaceService (decision-making): hot/warm/cold deliberation traces
   "Bootstrap dependency" model: ElevenLabs/EfficientNet/LLM seed the cache,
   then Type 1 retrieval replaces the slow path.

7. PRINCIPLE: Determinism via Seeded Mulberry32
   All Xavier projections use named hex seeds (0xf05e fusion, 0xd41e drives,
   0xa1de0 video, 0xa0d10 audio, 0xface0 face, 0x5ce0e scene) so embeddings
   are reproducible across restarts despite no training.

8. PRINCIPLE: Verbatim Telemetry (No LLM Summarization)
   InnerMonologuePanel + MaintenanceLogsPanel + SystemLogsPanel all show raw
   TimescaleDB events. The store builds entries from event fields directly.
   Frontend never re-paraphrases through LLM.

9. PRINCIPLE: Guardian Asymmetry
   Guardian feedback weighting (×2 confirmation, ×3 correction) is hardcoded
   in drive-engine + applyGuardianWeight pure function in shared.
   Supervisor verdicts can flag for guardian, but guardian-source interventions
   are first-class. Only guardian (via dashboard) can promote proposed_drive_rules
   to active drive_rules.

10. PRINCIPLE: Working Memory as Activation Selection (Not Storage)
    WorkingMemoryService doesn't store; it selects from existing knowledge
    using 5-signal activation (relevance + source confidence + recency +
    drive modulation + spreading). Hot residual layer 30s TTL.

11. PRINCIPLE: Cognitive Cycle as Two-Track (Tensor Shadow + LLM Reality)
    Shadow mode: tensor model receives every cycle's data + sees its argmax
    compared against the LLM's choice for agreement scoring.
    Audit: still LLM-driven but agreement tracked.
    Partial: tensor takes over for graduated categories (max conf 0.79 → forces Type 2).
    Full: tensor decides everything (max conf 0.95).
    Panel divergence > 0.3 caps ALL candidates < 0.80 → forces Type 2 for
    sanity check.

12. PRINCIPLE: 12-Dimensional Affective Substrate
    Not configurable, not extensible at runtime — DriveName enum is fixed.
    SystemHealth is a derived composite. Each drive has hand-tuned accumulation
    rate calibrated to minutes-not-seconds dynamics. Cross-modulation is 5
    typed rules in fixed priority order.

13. PRINCIPLE: Three Knowledge Graphs in Strict Separation
    WKG (world facts, large), SKG (Sylphie's self-model, small), OKG (person
    models per User.id). NO cross-instance queries. Bootstrap creates parallel
    CoBeing anchors in WKG + SKG.

14. PRINCIPLE: PKG (Codebase Graph) is Separate from Cognition
    Fourth Neo4j instance, separate port. Used by Claude Code via 7 MCP tools.
    Has its own seed/sync pipelines, different from Learning's KG writers.

15. PRINCIPLE: Cycle Cadence as Cognitive Constraint
    Drive tick: 1Hz (1000ms). Learning maintenance: 60s. Reflection: 5min.
    Cross-session synthesis: 30min. Confidence decay: 10min. Planning: 30s.
    Telemetry broadcast: 2Hz. Sensory log: 2Hz. These are deliberate "minutes
    not milliseconds" pacing for a motivational system, not perf concessions.

16. EMPTY SHELLS / FUTURE WORK (revealed by code reading):
    - EWC in cognition: stub (uniform Fisher, never anchored)
    - per_category_confidence dict: never written
