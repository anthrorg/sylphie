# 2026-03-29 -- E8-T007 & E8-T010: PlanProposalService and PlanEvaluationService

## Changes

- NEW: src/planning/proposal/plan-proposal.service.ts -- Implements propose() and revise() methods for plan proposal generation from research and simulation outputs. Generates UUIDs for proposal IDs, creates action sequences with drive effects, builds abort conditions, and emits PROPOSAL_GENERATED events to event backbone.

- NEW: src/planning/evaluation/plan-evaluation.service.ts -- Implements evaluateExecution() for evaluating plan execution outcomes. Compares predicted vs actual drive effects, computes MAE, updates WKG node confidence via ACT-R dynamics, supports guardian feedback weighting, and emits PLAN_EVALUATION and PLAN_FAILURE events.

## Wiring Changes

- PlanProposalService: Injects ConfigService, IEventService (EVENTS_SERVICE), IDriveStateReader (DRIVE_STATE_READER)
- PlanEvaluationService: Injects ConfigService, IEventService, IWkgService, IDriveStateReader
- Both use createPlanningEvent() builder with 'any' cast workaround for TypeScript type inference limitation on discriminated union builders

## Known Issues

- queryRecentMAEs() in PlanEvaluationService returns empty array placeholder. Proper implementation requires TimescaleDB query for PLAN_EVALUATION events. Graduation/demotion eligibility checks are functional but will always receive zero recent MAEs until this is implemented.
- extractExpectedEffects() and extractACTRParams() use 'any' type casts due to procedure node shape being unknown at compile time.

## Gotchas for Next Session

- The type signature for createPlanningEvent has a known TypeScript limitation: discriminated union type narrowing doesn't work perfectly in all contexts, requiring '(createPlanningEvent as any)' casts. This matches the pattern in procedure-creation.service.ts and is the project's standard workaround.
- MAE computation correctly handles Partial<Record<DriveName, number>> by iterating over union of both expected and actual drive keys, falling back to 0 for missing keys.
- PlanEvaluationService relies on IWkgService.recordRetrievalAndUse() for ACT-R updates rather than manual parameter manipulation, which correctly integrates with the confidence service.
