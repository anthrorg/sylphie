# 2026-03-30 — Epic 9 T006/T007: GraphController & ConversationController

## Summary
Implemented two REST API controllers for WKG visualization and conversation history querying.

## Changes
- NEW: `src/web/controllers/graph.controller.ts` — WKG read-only visualization API
  - GET `/api/graph/snapshot` — paginated graph snapshot with optional node-centered query
  - GET `/api/graph/stats` — aggregate WKG statistics (node/edge counts, provenance distribution)
  - GET `/api/graph/subgraph` — neighborhood query centered on a specific node
  - All responses include provenance and ACT-R confidence scores per CANON §Provenance Is Sacred
  - Enforces depth/maxNodes limits from WebConfig (maxDepth=3, maxNodes=200)
  - Query timeout support (default 5000ms) prevents long-running graph traversals

- MODIFIED: `src/web/controllers/conversation.controller.ts` — conversation history REST API
  - GET `/api/conversation/history` — retrieve paginated conversation history with pagination
  - GET `/api/conversation/messages/:conversationId` — retrieve all messages for a specific session/correlation
  - Extracts communication events (INPUT_RECEIVED, RESPONSE_GENERATED, etc.) from TimescaleDB
  - Converts events to ConversationMessage DTOs with drive snapshots and theater check results
  - Includes DriveSnapshot→DriveSnapshotDto conversion helper for DTO serialization

## Wiring Changes
- Both controllers inject IWkgService (WKG_SERVICE token) and IEventService (EVENTS_SERVICE token)
- GraphController reads WebConfig.graphVisualization for query limits
- Both record telemetry events to TimescaleDB for observability

## Implementation Details
- Uses `computeConfidence()` from shared/types/confidence.types to compute dynamic ACT-R values
- GraphController extracts human-readable labels from node properties (name, label, title fallback)
- ConversationController groups events by correlationId and sorts chronologically
- Pagination via validatePaginationParams() utility (default limit=50, max=500)
- All query errors caught and returned as InternalServerErrorException (no raw DB errors exposed)

## Known Issues
- None. Both controllers pass full TypeScript type-check.

## Gotchas for Next Session
- DriveSnapshot ≠ DriveSnapshotDto: conversation controller includes conversion helper
- SylphieEvent payload field is optional; use payload?.field ?? defaultValue pattern
- Graph query timeout uses Promise.race() not AbortController (NestJS compatibility)
- Empty graph queries return valid empty responses, not errors (per spec)
